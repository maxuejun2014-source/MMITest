# 手机工厂测试应用开发文档 v3.0

## 更新说明
根据用户完善的需求进行更新，主要变更：
1. 明确主界面包含四个按钮：自动测试、单项测试、测试结果、退出
2. 所有界面全屏显示，隐藏状态栏和导航栏，通过BaseActivity实现
3. 测试项统一在配置文件中管理，便于增删
4. 每个测试项对应独立的Activity文件
5. 明确21个测试项列表和具体要求
6. 单个测试项包含PASS和False按钮，部分测试项添加防呆设计

## 项目概述

### 项目背景
开发一款Android应用，用于工厂环境下测试手机各项基本功能。该应用将提供单项测试和自动测试两种模式，帮助工厂质检人员快速验证手机硬件功能是否正常。

### 项目目标
1. 提供直观的用户界面，便于工厂操作人员使用
2. 实现手机各项硬件功能的测试
3. 支持单项测试和自动测试两种模式
4. 保存测试结果到本地，便于追溯和分析
5. 提供清晰的测试结果展示和报告

## 功能需求

### 1. 主界面设计
- **四个主要功能按钮**：
  1. **自动测试**：进入自动测试流程
  2. **单项测试**：进入单项测试选择界面
  3. **测试结果**：查看历史测试结果
  4. **退出**：退出应用程序

- **设计要求**：
  - 大按钮设计，便于戴手套操作
  - 简洁直观，工厂环境使用
  - 全屏显示，隐藏状态栏和导航栏

### 2. 界面显示要求
- **全屏显示**：所有Activity均需全屏显示，隐藏状态栏和导航栏
- **基础类**：创建BaseActivity实现全屏显示功能，所有Activity继承BaseActivity
- **一致性**：所有界面保持统一的视觉风格和交互模式

### 3. 测试项配置管理
- **配置文件**：在单独的文件中维护测试项列表（如`TestConfig.kt`）
- **便于管理**：增删测试项只需修改配置文件
- **结构清晰**：每个测试项包含：ID、名称、描述、对应的Activity类、是否启用等
- **可扩展性**：支持后续添加新的测试项

### 5. 单项测试功能
- **独立Activity**：每个测试项对应独立的Activity文件（如`SimTestActivity.kt`）
- **测试界面**：显示测试说明、操作指引和实时状态
- **控制按钮**：每个测试界面必须包含PASS和False按钮
- **防呆设计**：某些测试项添加防呆设计，防止误操作
  - 例如：LCD测试需要连续显示多个颜色，防止误判
  - 例如：按键测试需要按特定顺序操作
- **实时反馈**：测试过程中提供实时状态反馈和提示

### 6. 自动测试功能
- **顺序执行**：按照配置文件中的顺序自动执行所有测试项
- **智能中断**：某个测试失败时可选择继续或停止
- **进度显示**：实时显示测试进度和当前测试项
- **结果汇总**：自动生成完整的测试报告
- **批量处理**：支持批量测试多台设备

### 7. 测试结果管理
- **本地存储**：将测试结果保存到本地文件（JSON格式）
- **结果查看**：查看历史测试记录，按时间、设备、通过率等筛选
- **报告导出**：支持导出测试报告为PDF或Excel格式
- **统计功能**：统计测试通过率、常见问题、设备趋势等

## 技术架构

### 开发环境
- **开发语言**：Kotlin
- **最低API级别**：Android 8.0 (API 26)
- **目标API级别**：最新稳定版
- **开发工具**：Android Studio
- **构建工具**：Gradle
- **UI框架**：Jetpack Compose

### 架构模式
采用MVVM（Model-View-ViewModel）架构：
- **数据层**：测试逻辑、硬件访问、数据存储
- **视图模型层**：管理UI状态、处理用户交互
- **视图层**：Activity、Fragment、Compose界面

### 主要技术组件
1. **界面框架**：Jetpack Compose
2. **基础类**：BaseActivity（全屏显示实现）
3. **数据存储**：Room数据库（测试结果）、SharedPreferences（配置）
4. **权限管理**：Android权限系统（动态权限申请）
5. **硬件访问**：Android硬件API、传感器API
6. **异步处理**：Kotlin协程
7. **依赖注入**：Hilt
8. **测试框架**：JUnit、Espresso

### 1. BaseActivity全屏显示实现
```kotlin
import androidx.activity.ComponentActivity
import androidx.core.view.WindowCompat
import androidx.core.view.WindowInsetsCompat
import androidx.core.view.WindowInsetsControllerCompat

abstract class BaseActivity : ComponentActivity() {
    override fun onCreate(savedInstanceState: Bundle?) {
        super.onCreate(savedInstanceState)
        enableFullScreen()
    }

    protected fun enableFullScreen() {
        // 隐藏状态栏和导航栏
        WindowCompat.setDecorFitsSystemWindows(window, false)

        // 设置沉浸式模式
        val windowInsetsController = WindowCompat.getInsetsController(window, window.decorView)
        windowInsetsController.hide(WindowInsetsCompat.Type.systemBars())
        windowInsetsController.systemBarsBehavior =
            WindowInsetsControllerCompat.BEHAVIOR_SHOW_TRANSIENT_BARS_BY_SWIPE
    }

    override fun onWindowFocusChanged(hasFocus: Boolean) {
        super.onWindowFocusChanged(hasFocus)
        if (hasFocus) {
            enableFullScreen()
        }
    }
}
```

## 防呆设计要点
1. **LCD测试**：需要按顺序显示多个颜色（红→绿→蓝→白→黑），每个颜色显示3秒，防止快速跳过
2. **按键测试**：需要按特定顺序操作按键（如：音量+ → 音量- → 电源键）
3. **TP测试**：需要完成特定手势（画圆、画线等）才能继续
4. **摄像头测试**：需要实际拍照/录像并确认图像质量
5. **音频测试**：需要播放声音并确认可以听到
6. **GPS测试**：需要等待定位成功（搜星≥3）才能通过

## 测试策略
1. **单元测试**：测试各个硬件模块的独立功能
2. **集成测试**：测试完整测试流程和数据流
3. **UI测试**：测试界面交互和用户体验
4. **真机测试**：在不同型号手机上测试兼容性
5. **压力测试**：长时间运行测试稳定性
6. **边界测试**：测试各种异常情况和边界条件
